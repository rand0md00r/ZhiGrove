# 🔄 ZhiGrove 工作流指南

> **从信息到知识的完整流程**

## 📅 每日工作流（5-10分钟）

### 🌅 早晨（2分钟）
- [ ] 查看收件箱，快速浏览新内容
- [ ] 标记今天要处理的重要事项
- [ ] 更新今日学习目标

### 🌙 睡前（3-5分钟）
- [ ] 记录今天学到的新知识
- [ ] 整理收件箱中的内容
- [ ] 快速分类到对应目录

**快速分类规则：**
- 论文/技术 → `20-papers/`
- 想法/灵感 → `30-ideas/`
- 实验/代码 → `40-experiments/`
- 总结/报告 → `50-reports/`

## 📊 每周工作流（30-60分钟）

### 🗓️ 周日整理时间
- [ ] **收件箱清理**（15分钟）
  - 处理一周积累的临时文件
  - 删除重复和无用内容
  - 合并相关内容

- [ ] **内容关联**（20分钟）
  - 检查新内容与已有知识的关联
  - 更新相关文档的链接
  - 添加标签和关键词

- [ ] **知识更新**（15分钟）
  - 更新 `10-knowledge/` 中的领域知识
  - 检查是否有新的知识结构
  - 更新README中的近期更新

- [ ] **下周计划**（10分钟）
  - 设定下周学习目标
  - 安排实验和调研计划
  - 更新TODO列表

## 📈 每月工作流（2-3小时）

### 🗓️ 月末深度整理
- [ ] **知识体系梳理**（1小时）
  - 检查各领域知识的完整性
  - 识别知识空白和重复
  - 重新组织知识结构

- [ ] **内容质量检查**（1小时）
  - 检查文档的完整性和准确性
  - 更新过时的信息
  - 补充缺失的细节

- [ ] **工具和流程优化**（30分钟）
  - 检查脚本是否正常工作
  - 优化模板和流程
  - 添加新的自动化功能

- [ ] **月度报告**（30分钟）
  - 总结本月学习成果
  - 分析知识增长趋势
  - 制定下月学习计划

## 🎯 关键原则

### 1. **渐进式整理**
- 不要试图一次性完美
- 先有结构，再完善内容
- 持续迭代和改进

### 2. **关联性优先**
- 新知识要与已有知识建立联系
- 使用标签和链接建立知识网络
- 避免知识孤岛

### 3. **实用性导向**
- 记录是为了使用
- 定期回顾和应用
- 删除无用的内容

### 4. **自动化优先**
- 能用脚本的不用手动
- 建立标准化的流程
- 减少重复性工作
